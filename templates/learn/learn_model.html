Security Models

What is a security model?
  • Scheme that specifies and enforces security policies (E.g. Reference monitors)
  • Provides rules of the road for security in operating systems
The 4 approaches to implement a security model:
  1. No security (Security "as is", default)
    • Use default security configuration
    • Does not have addtional security protection
  2. Security through obscurity (Hiding confidential information)
  3. Host security
    • Provide security by protecting the host
    • E.g. Provide user authentication and access control
  4. Network security (E.g. Encrypting communication channel)
