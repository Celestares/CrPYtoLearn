Cryptography Concepts and Techniques

Shift Cipher
It shifts each character in the plaintext by x amount (specified by user as the key) to the right

Credit to Wikipedia

Example : If the key is 3, all character with 'a' will shift (become) to 'd' as ciphertext, 'b' become 'e', 'c' become 'f' and so on.

Mono-alphabet Cipher

Similar to shift cipher, but uses random substitution, meaning each "A" is replaced by any character from "B" to "Z", each "B" is replaced by any character with "A" or "C" to "Z" and so on.


Credit to Mohd Arif slides from slideshare.net

Example : All "A" will be replaced by "D", all "B" will be replaced by "K" and so on.

Rail Fence Technique

Writes the plaintext is diagonals to form rows (row specified by user) and is read row by row


Credit to ResearchGate

Example : In the above picture, 3 rows were created. Since it is read row by row, 1st row is read as "Hor", 2nd row is read as "el ol", and last row is read as "lWd". Combining the rows together would result in "Horel ollWd

Simple Columnar Transposition Technique

Similar to rail fence technique but plaintext is written in rows is read column by column

Note : Does not have to be read by column 1 then column 2 then column 3 and so on.. a key can specify which column to read first

Let the key be "car", it will read the column based in alphabet order, meaning since "a" of the key "car" is the 1st letter of the alphabet, the 2nd column will be read first, followed by 1st column since "c" is the 3rd letter and then the last column since "r" is the 18th alphabet.


Credit to Crypto Corner

Example : Input plaintext is "THETOMATOISAPLANTINTHENIGHTSHADEFAMILY" and the key is "TOMATO", since "A" is the 1st letter of the alphabet and appear in the 4th letter of the key, the 4th column is read first.

Vernam Cipher

It treats each plaintext character and key character as a number and the numbers are added up and converted back to text (ciphertext)

A = 0, B = 1, C =2 and so on.


Taken from Applied Cryptography lecture notes

Example : Input plaintext is "HELLO" and key is "WORLD", both are converted to numbers and added up if the number is greater than 26, it will be subtracted by 26. The output number will then be converted back to text.

Diffie-Hellman Key Exchange

A secret key can be agreed upon without any secrets being exchanged


Taken from Applied Cryptography lecture notes

Example : Two people will agree on 2 large prime number (can be known) and both of them are to decide on any secret number for each of themselves and calculate g^x mod n where x is the secret number they decided on. They exchange their calculated value and apply the same calculation except g is replaced by the calculated value they have gotten from their partner. This results in them receving the same result without needing to pass their secret number to each other.
